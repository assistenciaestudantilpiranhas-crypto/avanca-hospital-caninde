# GSI Design System

O GSI Design System (GDS) e o produto oficial de design da GSI HealthTech para padronizar interfaces, componentes, experiencia clinica, acessibilidade, movimento e orientacao de agentes de IA no ecossistema GSI.

## Relacao com o ecossistema

- GSI HealthTech: ecossistema institucional e marca guarda-chuva.
- GSI ONE: plataforma digital de saude.
- Avanca Hospital: programa de implementacao e caso hospitalar.
- GHAES: padrao de engenharia, seguranca, rastreabilidade e governanca para evolucao assistida por IA.

O GDS organiza a linguagem visual e interativa usada pelo GSI ONE e por implementacoes como o Avanca Hospital, preservando consistencia entre produto, operacao hospitalar, gestao publica, auditoria e indicadores.

## Papel do GDS

O GDS orienta:

- identidade visual;
- tokens de design;
- componentes reutilizaveis;
- UX clinica;
- acessibilidade;
- motion e transicoes;
- uso seguro de IA no desenvolvimento de interfaces.

## Uso por agentes de IA

Agentes como Codex, Claude Code e ChatGPT devem consultar estes documentos antes de criar ou alterar interfaces. Alteracoes visuais devem preservar IDs, eventos, funcoes, regras de negocio, fluxos assistenciais, integracoes, Supabase e localStorage, exceto quando houver autorizacao explicita para modifica-los.
